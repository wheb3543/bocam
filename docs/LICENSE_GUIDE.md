# دليل تراخيص مشروع BOCAM CRM

![License](https://img.shields.io/badge/license-Dual--Licensed-blue.svg)
![Type](https://img.shields.io/badge/type-MIT%20%26%20Proprietary-orange.svg)

مرحباً بك في دليل تراخيص منصة **BOCAM CRM**. يعتمد هذا المشروع على نموذج ترخيص مزدوج يفصل بشكل صارم وقانوني بين المكونات المفتوحة المصدر والمكونات التجارية المحمية. يهدف هذا الدليل إلى توضيح حدود هذا الفصل والتعليمات التقنية الخاصة بنظام الترخيص.

## ⚖️ الفصل القانوني والتراخيص (Legal Separation)

ينقسم المشروع إلى جزأين رئيسيين متميزين قانونياً وتقنياً:

### 1. الأجزاء المفتوحة المصدر (Open-Source Core)
* **الترخيص:** رخصة **MIT** (راجع ملف `LICENSE` في جذر المشروع).
* **المكونات المشمولة:**
  - الهيكل الأساسي للواجهة الأمامية (Frontend Core) وإعدادات بيئة التطوير.
  - الواجهات العامة والصفحات التعريفية (مثل الصفحة الرئيسية العامة، صفحات الأطباء، وإدارة العروض العامة).
  - الهيكل القاعدي والخدمات الأساسية غير المرتبطة بالوحدات المتقدمة.
* **الاستخدام:** يُسمح باستخدام هذه الأجزاء وتعديلها وتوزيعها مجاناً وبدون قيود، بشرط تضمين إشعار حقوق النشر والرخصة الأصلي.

### 2. الأجزاء المحمية والتجارية (Proprietary & Protected SaaS Modules)
* **الترخيص:** **ترخيص تجاري مغلق مملوك ومحمي (Proprietary License)** لصالح شركة **IdeaHub** وتحت إدارة **BOCAM**.
* **المكونات المشمولة:**
  - **نظام واتساب المتكامل (14 صفحة):** بما يشمل إدارة المحادثات المباشرة، البث الجماعي (Broadcast)، لوحة التحكم بالرسائل والقوالب، والربط مع WhatsApp Business Cloud API.
  - **نظام التقارير والإحصائيات المتقدم (4 صفحات):** بما يشمل تقارير الإيرادات، تحليلات أداء الحملات، إحصائيات المرضى، والرسوم البيانية التفاعلية.
  - **بوابة المرضى المتكاملة (10 صفحات):** نظام تسجيل الدخول بـ OTP وعرض النتائج الطبية (المختبر والأشعة) وإدارة المواعيد المتقدمة.
  - **نظام الحماية والترخيص الآلي:** آليات التحقق من سلامة النظام وتشفير التراخيص.
* **الاستخدام:** يمنع منعاً باتاً نسخ أو توزيع أو تعديل أو إعادة هندسة أي من هذه الوحدات البرمجية دون الحصول على ترخيص رسمي معتمد وموقع رقمياً من الجهات المالكة. يخضع هذا الجزء لنظام الحماية النشط والتعطيل التلقائي (Kill Switch).

## 🔐 نظام الترخيص المحلي المشفر (SaaS License System)

تم بناء نظام الترخيص لحماية الملكية الفكرية وضمان تشغيل المنصة في بيئات معزولة ومحددة مسبقاً.

### المميزات التقنية للنظام:
* **التشفير:** استخدام خوارزمية **RSA-2048** مع توقيع رقمي **SHA-256**.
* **الربط العتادي:** ربط الترخيص بالمعرف العتادي للجهاز (**Hardware ID**).
* **التشغيل المستقل:** يعمل بالكامل دون الحاجة للاتصال بشبكة الإنترنت (Air-gapped Environments).
* **الحماية ضد التلاعب:** نظام مراقبة صامت (Heartbeat) وحماية ضد التلاعب بمرور الوقت وتغيير ساعة النظام (Anti-Clock-Tampering).

## 🛠️ دليل تشغيل وإدارة نظام الترخيص (للمطورين والمسؤولين)

يتضمن هذا القسم التعليمات والأوامر اللازمة لإعداد وتوليد التراخيص للعملاء:

### 1. الحصول على معرف الجهاز (Hardware ID):
لربط الترخيص بجهاز العميل المستهدف، يجب تشغيل الأمر التالي على خادم العميل لاستخراج معرف العتاد الفريد:

```bash
pnpm license:get-hardware-id
```
*مثال على مخرج الأمر:* `086D41DCC716`

### 2. توليد مفاتيح التشفير (RSA Keys):
لتوليد زوج المفاتيح العام والخاص الجديد (يتم تنفيذ هذا الأمر من قبل الشركة المالكة فقط):

```bash
pnpm license:generate-keys
```

### 3. توليد ترخيص للعميل (Generate License):
أولاً، قم بحساب تاريخ انتهاء الترخيص بصيغة Unix Timestamp (على سبيل المثال، ترخيص لمدة سنة واحدة):

```bash
node -e "console.log(Math.floor((new Date().getTime() + 365 * 24 * 60 * 60 * 1000) / 1000))"
```

ثم قم بتشغيل أمر التوليد بتمرير معرف جهاز العميل، ووقت الانتهاء، والميزات المراد تفعيلها:

```bash
pnpm license:generate <HARDWARE_ID> <TIMESTAMP> "<FEATURES>"
```
*مثال:*
```bash
pnpm license:generate 086D41DCC716 1781677061 "*"
```

### 🎯 الميزات المتاحة للتحكم (Feature Flags):
يمكنك تفعيل ميزات محددة للعميل عبر تمريرها مفصولة بفاصلة، أو تمرير `*` لتفعيل كل شيء:
* `*` - تفعيل جميع ميزات النظام بلا استثناء.
* `whatsapp` - نظام واتساب وإدارة المحادثات.
* `reports` - نظام التقارير والإحصائيات المتقدمة.
* `patient_portal` - بوابة المريض للنتائج والمواعيد.
* `camps` - إدارة المخيمات الطبية.
* `offers` - إدارة العروض والخصومات الطبية.

## 🛡️ آليات الأمان المتقدمة ونظام التعطيل (Kill Switch)

لحماية المكونات التجارية، يحتوي النظام على طبقات حماية مدمجة:
1. **التحقق المستمر:** يقوم النظام بالتحقق دورياً من صحة ملف `license.json` والتوقيع الرقمي.
2. **التعطيل التلقائي (Kill Switch):** في حال اكتشاف تلاعب في ملف الترخيص أو استخدام مفتاح غير صالح، سيقوم النظام تلقائياً بإيقاف خادم التطبيق وعرض شاشة قفل أمنية.
3. **مكافحة التلاعب بالوقت:** يمنع النظام تشغيل التطبيق في حال تقديم ساعة النظام إلى الخلف للالتفاف على تاريخ انتهاء الترخيص.
