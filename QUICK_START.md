# ⚡ دليل التشغيل السريع للمطورين (BOCAM CRM Quick Start Guide)

مرحباً بك في دليل التشغيل السريع لـ **BOCAM CRM**. تم تصميم هذا الدليل لتمكين المطورين وفرق العمل من تشغيل بيئة تطوير محلية متكاملة وسريعة خلال **أقل من دقيقتين** باستخدام تقنيات Docker وحزمة المحاكاة الذكية للخدمات (Mock Mode).

---

## 📦 المتطلبات الأساسية (Prerequisites)

قبل البدء، تأكد من تثبيت الأدوات التالية على جهازك:
1. **Docker & Docker Compose** (لإدارة قواعد البيانات والخدمات).
2. **Node.js** (الإصدار 18 أو أحدث).
3. **pnpm** (مدير الحزم السريع - لتثبيته: `npm install -g pnpm`).

---

## 🚀 خطوات التشغيل بلمحة بصر

اتبع الخطوات الأربع البسيطة التالية لبدء التشغيل:

### 1️⃣ تهيئة ملف الإعدادات السريعة
قم بنسخ ملف الإعدادات المخففة والمجهزة مسبقاً بالتسمية الصحيحة:
```bash
cp .env.example.quick .env
```
*ملاحظة: هذا الملف يحتوي على تفعيل افتراضي لـ `MOCK_MODE=true` ليتيح لك تشغيل النظام بالكامل محلياً وبدون الحاجة لربطه الفعلي بحسابات WhatsApp Business أو Twilio أو SendGrid.*

### 2️⃣ تشغيل الخدمات المساعدة عبر Docker
نوصي بتشغيل خادم قاعدة البيانات (MySQL) والذاكرة المؤقتة (Redis) عبر Docker لتفادي التثبيت اليدوي المعقد.

شغّل الأمر التالي لتشغيل الحاويات في الخلفية:
```bash
# لتشغيل قاعدة بيانات MySQL المجهزة بالجداول التلقائية
docker-compose -f deploy/docker-compose.yml up -d mysql
```
*(اختياري) لتشغيل خادم الذاكرة المؤقتة Redis (إذا تم تفعيله):*
```bash
docker-compose up -d redis
```

### 3️⃣ تثبيت الحزم وإعداد قاعدة البيانات
قم بتثبيت الاعتماديات الخاصة بالمشروع، ثم ادفع بهيكل الجداول وبيانات التجربة الأولية لقاعدة البيانات:
```bash
# 1. تثبيت الاعتماديات
pnpm install

# 2. إنشاء وهيكلة جداول قاعدة البيانات (40+ جدول)
pnpm db:push

# 3. ملء قاعدة البيانات ببيانات تجريبية أولية (أطباء، حجوزات، حملات)
pnpm db:seed
```

### 4️⃣ تشغيل التطبيق في وضع التطوير (Hot Reload)
الآن، أنت جاهز لتشغيل خادم التطبيق المحلي (الواجهة الأمامية والخلفية معاً):
```bash
pnpm dev
```

## 🚪 روابط الدخول والتجربة المباشرة

بمجرد تشغيل التطبيق، يمكنك تصفح الأنظمة المختلفة عبر الروابط التالية:

* **🏠 الواجهة العامة للموقع:** `http://localhost:3000`
  * تصفح الأقسام العامة، استعراض العروض الطبية، والمخيمات المتاحة.
* **💼 لوحة التحكم الإدارية (Admin Dashboard):** `http://localhost:3000/admin`
  * بوابة إدارة علاقات العملاء، إدارة المواعيد والحجوزات، إطلاق الحملات التسويقية، ونظام المهام.
  * *لتسجيل الدخول السريع: اضغط على زر تسجيل الدخول (سيتم توجيهك للمحاكاة تلقائياً).*
* **📱 بوابة المريض (Patient Portal):** `http://localhost:3000/patient`
  * استعرض سجلاتك الطبية الافتراضية وحجوزاتك السابقة.
* **🛠️ إدارة قاعدة البيانات (Drizzle Studio):**
  ```bash
  pnpm db:studio
  ```
  * يفتح لك واجهة رسومية لإدارة وتصفح بيانات الجداول مباشرة عبر المتصفح على الرابط `http://localhost:4983`.

## 🎭 مميزات بيئة التطوير السريعة (Mock Mode)

عند تفعيل وضع المحاكاة في ملف `.env` عبر `MOCK_MODE=true`، فإن التطبيق يتميز بالآتي:
* **مستكشف أخطاء ذكي:** لا تظهر أي أخطاء عند إرسال رسائل الواتساب أو البريد؛ بل يتم تسجيلها في سجلات السيرفر كعمليات محاكاة تامة بنجاح.
* **تجاوز بوابات المصادقة الصعبة:** نظام تسجيل الدخول محاكى محلياً، مما يسهل تجربة لوحة الإدارة بأي بريد إلكتروني تجريبي.
* **سرعة فائقة:** تشغيل واختبار كامل المنصة محلياً دون انتظار استجابة السيرفرات الخارجية أو استهلاك رصيد الرسائل.

## 🔍 استكشاف الأخطاء الشائعة (Troubleshooting)

* **فشل الاتصال بقاعدة البيانات (`ECONNREFUSED`):**
  * تأكد من أن حاوية MySQL تعمل بشكل صحيح عبر الأمر `docker ps`.
  * تحقق من أن منفذ `3306` ليس مشغولاً بخادم MySQL محلي مثبت سابقاً على جهازك.
* **أخطاء الترخيص عند بدء التشغيل:**
  * يدعم النظام محاكاة الترخيص تلقائياً في وضع التطوير المحلي. في حال طلب النظام ترخيصاً، قم بتوليد ترخيص تجريبي لتجربتك المحلية باتباع التعليمات في [دليل التراخيص (docs/LICENSE_GUIDE.md)](docs/LICENSE_GUIDE.md).
