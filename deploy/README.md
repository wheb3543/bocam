## نظرة عامة

BOCAM CRM Platform هو نظام إدارة علاقات عملاء (CRM) متكامل للقطاع الطبي، مصمم كمنصة SaaS قائمة على الترخيص المحلي المشفر. يوفر النظام:
- إدارة المرضى والعلاج
- نظام واتساب متكامل (14 صفحة)
- نظام التقارير والإحصائيات (4 صفحة)
- بوابة المرضى (10 صفحة)
- إدارة المخيمات والعروض
- نظام حماية متقدم متعدد الطبقات

## المميزات الرئيسية

### 🔐 نظام الترخيص المحلي
- تشفير RSA-2048 بتوقيع رقمي SHA-256
- ربط الترخيص بـ Hardware ID
- يعمل بدون اتصال إنترنت
- Kill Switch لحماية الأمان

### 💓 نظام Heartbeat الصامت
- إرسال بيانات الأمان المجهولة كل 24 ساعة
- حماية ضد التلاعب بالوقت (Clock Tampering)
- التحقق من سلامة النظام عن بعد

### 🎯 Feature Flags متقدمة
- التحكم في الميزات على مستوى API و UI
- حماية الطرق (Route Protection)
- صفحات قفل الميزات مع رسائل ترقية

### 🚀 أداء عالي
- بناء على React + TypeScript
- Drizzle ORM لقواعد البيانات
- Vite للبناء السريع

## المتطلبات

- **Node.js:** v18.0.0 أو أحدث
- **MySQL:** 8.0+ أو TiDB Cloud (موصى)
- **RAM:** 4GB (موصى 8GB+ للإنتاج)
- **Storage:** 20GB SSD (موصى 50GB+ للإنتاج)

## التثبيت السريع

## ⚖️ نظام الترخيص والفصل القانوني

يعتمد مشروع BOCAM CRM على نظام ترخيص مزدوج يفصل بين المكونات المفتوحة المصدر (المحمية برخصة MIT) والمكونات البرمجية التجارية المتقدمة (المملوكة لـ IdeaHub).

* **الأجزاء المفتوحة:** الهيكل العام والمكونات الأساسية (رخصة MIT).
* **الأجزاء المحمية:** نظام واتساب (14 صفحة)، نظام التقارير والإحصائيات المتقدم (4 صفحات)، بوابة المرضى المتكاملة (10 صفحات)، ونظام الأمان والتشغيل المحلي.

لقد تم نقل تفاصيل نظام الترخيص، كيفية توليد المفاتيح، والتحقق العتادي إلى الدليل المخصص. يرجى مراجعة الدليل الكامل للتراخيص والأمان عبر الرابط التالي:
👉 **[دليل التراخيص والأمان المخصص (docs/LICENSE_GUIDE.md)](../docs/LICENSE_GUIDE.md)**

## الأمان

### طبقات الحماية:

1. **ترخيص محلي مشفر** - RSA-2048 + SHA-256
2. **Kill Switch** - يوقف السيرفر إذا تم التلاعب
3. **Heartbeat** - مراقبة عن بعد كل 24 ساعة
4. **Anti-Clock-Tampering** - حماية ضد تغيير الوقت
5. **Feature Middleware** - حماية على مستوى API

### اختبار الحماية:

```bash
# اختبار Kill Switch
# غيّر حرفًا واحدً في license.json
# أعد تشغيل السيرفر
pnpm dev
# النتيجة: Kill Switch ACTIVATED
```

## استكشاف الأخطاء

### السيرفر لا يعمل:
```bash
# فحص المنافذ
lsof -i :3000

# فحص السجلات
tail -f server.log

# إعادة التشغيل
pnpm start
```

### خطأ في الترخيص:
```bash
# التحقق من وجود license.json
ls -la license.json

# استعادة نسخة احتياطية
cp license.json.backup license.json
```

### خطأ في قاعدة البيانات:
```bash
# فحص حالة MySQL
sudo systemctl status mysql

# فحص الاتصال
mysql -u root -p bocam_crm
```

راجع [دليل التثبيت](docs/INSTALLATION_GUIDE.md#استكشاف-الأخطاء-الشائعة) للمزيد.
