# 🛠️ دليل الصيانة والعمليات التقنية لمشرفي النظام (BOCAM CRM DevOps & Maintenance Guide)

مرحباً بك في الدليل الشامل لعمليات الصيانة والتشغيل وإدارة البنية التحتية لمنصة **BOCAM CRM**. تم تصميم هذا الدليل لفرق الـ DevOps ومسؤولي النظام لإيضاح آلية عمل جميع السكربتات المؤتمتة المتوفرة في مجلد `deploy/` وكيفية إدارتها لضمان استقرار وحماية وأداء النظام في بيئة الإنتاج.

---

## 🗂️ نظرة عامة على هيكل أدوات العمليات (Operations Directory)

يحتوي مجلد `deploy/` على كافة الأدوات والملفات اللازمة لنشر وإدارة ومراقبة المنصة:
* **`deploy/scripts/`:** يحتوي على سكريبتات التثبيت وإعداد التراخيص العتادية.
* **`deploy/backup/`:** حزمة متكاملة للنسخ الاحتياطي (لقواعد البيانات والملفات) والرفع السحابي وتدوير النسخ القديمة.
* **`deploy/monitoring/`:** أدوات مراقبة أداء الخادم والموارد وقاعدة البيانات عبر Prometheus و Grafana.
* **`deploy/nginx/`:** إعدادات خادم الويب وتأمين النفق الرقمي وتوليد شهادات SSL وتجديدها تلقائياً.

---

## 🚀 القسم الأول: دليل التثبيت والتهيئة الآلية (`deploy/scripts/`)

### 📄 سكريبت التثبيت الأساسي (`install.sh`)
* **الوصف الفني:** سكريبت متكامل ومؤتمت بالكامل يقوم بتهيئة بيئة التشغيل بالكامل من نقطة الصفر، والتحقق من متطلبات العتاد والبرمجيات المتاحة، وتهيئة قاعدة البيانات، وتوليد مفاتيح التشفير والتراخيص المحلية.
* **المدخلات (Inputs):** 
  * قراءة ملف التكوين الافتراضي `.env.example` لتوليد `.env`.
  * التفاعل المباشر مع المسؤول لتأكيد التخطي في حال عدم وجود بعض المكونات غير الحرجة.
* **المخرجات (Outputs):**
  * تهيئة وتثبيت حزم الاعتماديات عبر `pnpm install`.
  * إنشاء ملف `.env` مشفر ومحمى مع توليد عشوائي آمن لـ `JWT_SECRET` عبر OpenSSL.
  * هيكلة قاعدة البيانات محلياً وتحديث الجداول عبر Drizzle ORM (`pnpm db:push`).
  * استخراج المعرف العتادي الفريد للجهاز (Hardware ID) اعتماداً على الماك أدرس الفعلي للشبكة.
  * توليد أزواج مفاتيح RSA-2048 العامة والخاصة وتخزينها في مجلد `license-keys/`.
  * توليد ملف ترخيص محلي تجريبي وصالح لمدة سنة كاملة في جذر المشروع باسم `license.json`.
* **أمثلة التشغيل الفعلي (Command Examples):**
  ```bash
  # تشغيل السكريبت بصلاحيات مناسبة لبدء التثبيت
  chmod +x deploy/scripts/install.sh
  ./deploy/scripts/install.sh
  ```

## 💾 القسم الثاني: نظام النسخ الاحتياطي والأرشفة الذكي (`deploy/backup/`)

تتضمن هذه الحزمة أربعة ملفات مترابطة لإدارة الحفظ الرقمي للملفات والبيانات وتأمينها خارج الخادم:

### 1️⃣ سكريبت النسخ الاحتياطي الأساسي (`backup.sh`)
* **الوصف الفني:** يقوم بأخذ لقطة كاملة لقاعدة بيانات MySQL وضغطها، وضغط كافة ملفات النظام الهامة، وحزم التراخيص، وملفات الإعدادات وتجميعها في مجلد يحمل تاريخ اليوم مع توليد ملف بيان (Manifest) تفصيلي.
* **المدخلات (Inputs):** إعدادات الملف الأساسي للتكوين الموجود في المسار `/etc/bocam-backup/config.sh`.
* **المخرجات (Outputs):**
  * قاعدة بيانات مضغوطة: `database_YYYYMMDD_HHMMSS.sql.gz`.
  * ملفات التطبيق (باستثناء المجلدات الثقيلة): `files_YYYYMMDD_HHMMSS.tar.gz`.
  * ملفات التراخيص والمفاتيح: `license_YYYYMMDD_HHMMSS.tar.gz`.
  * ملف الإعدادات البيئية: `env_YYYYMMDD_HHMMSS.tar.gz`.
  * ملف التقرير الفني: `manifest.json`.
* **أمثلة التشغيل الفعلي:**
  ```bash
  # لتشغيل النسخ الاحتياطي يدوياً في أي وقت
  ./deploy/backup/backup.sh
  ```

### 2️⃣ سكريبت تدوير النسخ القديمة وفحص السلامة (`cleanup-old-backups.sh`)
* **الوصف الفني:** يتأكد من الحفاظ على السعة التخزينية للخادم عبر حذف المجلدات التي تجاوزت عمر الاحتفاظ المحدد (مثلاً 30 يوماً). كما يتضمن وظيفة ذكية جداً للتحقق من سلامة وصلاحية النسخ الاحتياطية الأخيرة لضمان عدم وجود ملفات تالفة عبر اختبار التعبئة بـ `gzip -t`.
* **المدخلات:** متغير `RETENTION_DAYS` ومتغير `BACKUP_ROOT` من ملف التكوين.
* **المخرجات:** 
  * تفريغ مساحة القرص الصلب.
  * كتابة تقرير فحص تفصيلي للسلامة في المسار: `/var/log/bocam-backup/cleanup-report-YYYYMMDD.txt`.
* **أمثلة التشغيل الفعلي:**
  ```bash
  ./deploy/backup/cleanup-old-backups.sh
  ```

### 3️⃣ سكريبت الرفع السحابي الآمن (`upload-to-cloud.sh`)
* **الوصف الفني:** سكريبت مرن مخصص لنقل النسخ الاحتياطية المؤرشفة إلى خوادم التخزين السحابية الخارجية. يدعم السكريبت 3 أدوات رفع شهيرة تلقائياً: AWS CLI لـ S3، أداة Rclone لـ Cloudflare R2، وأداة MinIO Client (`mc`).
* **المدخلات:** مسار المجلد المراد رفعه (كوسيط إلزامي أثناء الاستدعاء).
* **المخرجات:** تزامن كامل للملفات المحددة على السحابة مع التخزين الآمن بمستويات الحفظ المطلوبة.
* **أمثلة التشغيل الفعلي:**
  ```bash
  # رفع مجلد النسخة الاحتياطية لليوم الحالي يدوياً إلى السحابة
  ./deploy/backup/upload-to-cloud.sh /var/backups/bocam/2026-07-05
  ```

### 4️⃣ ملف إعدادات النسخ الاحتياطي (`config.sh`)
* **الوصف الفني:** قالب إعدادات قياسي لربط بيانات الاتصال، وتفاصيل الحسابات السحابية والمسارات وصلاحيات الاحتفاظ. يجب نسخه إلى `/etc/bocam-backup/config.sh` قبل التشغيل.
* **أمثلة التهيئة:**
  ```bash
  # إنشاء مجلد الإعدادات الافتراضي ونقل القالب إليه
  sudo mkdir -p /etc/bocam-backup
  sudo cp deploy/backup/config.sh /etc/bocam-backup/config.sh
  # قم بتحرير الملف وضبط المتغيرات (بيانات المرور، ومستوى الضغط، والبريد الإلكتروني للتبليغات)
  sudo nano /etc/bocam-backup/config.sh
  ```

⏱️ **جدولة المهام الآلية (Automation via Cron):**
لإعداد عملية النسخ والتنظيف التلقائي يومياً عند الساعة **2:00 صباحاً**، يمكن استخدام التكوين الجاهز المرفق بملف `deploy/backup/bocam-backup.cron` أو إضافته لجدول النظام:
```bash
# افتح محرر كرون الخادم
sudo crontab -e

# أضف السطر التالي للتشغيل اليومي المؤتمت بالكامل
0 2 * * * /Users/applestore/Documents/GitHub/bocam/deploy/backup/backup.sh >> /var/log/bocam-backup/cron.log 2>&1
```

## 📊 القسم الثالث: نظام المراقبة وإدارة مؤشرات الأداء (`deploy/monitoring/`)

### 📄 سكريبت إعداد حزمة المراقبة (`setup-monitoring.sh`)
* **الوصف الفني:** يقوم بتشغيل حزمة المراقبة والتحليلات المتكاملة القائمة على حاويات Docker لتتبع استهلاك المعالج، والذاكرة، ومساحة التخزين، وأداء قواعد البيانات وحركة الشبكة.
* **المدخلات (Inputs):** متغيرات البيئة الاختيارية لتخصيص مسارات قواعد البيانات والذاكرة المؤقتة (`MYSQL_EXPORTER_DSN`, `REDIS_ADDR`).
* **المخرجات (Outputs):**
  * إنشاء المجلدات المحلية للمقاييس وحفظ البيانات.
  * تشغيل حزمة البرمجيات المترابطة في الخلفية عبر Docker Compose:
    * **Prometheus:** لجمع وتخزين مقاييس الأداء.
    * **Grafana:** لعرض لوحات التحكم والرسوم البيانية التفاعلية.
    * **Node Exporter:** لمراقبة عتاد الخادم الأساسي.
    * **cAdvisor:** لمراقبة الموارد المستهلكة من قبل حاويات Docker.
    * **MySQL Exporter / Redis Exporter / Nginx Exporter:** لمقاييس قواعد البيانات وخادم الويب والذاكرة.
    * **Alertmanager:** لإدارة التنبيهات وإرسال الإشعارات عند تخطي الحدود الآمنة.
* **أمثلة التشغيل الفعلي:**
  ```bash
  # تشغيل وإعداد حزمة المراقبة بالكامل
  chmod +x deploy/monitoring/setup-monitoring.sh
  ./deploy/monitoring/setup-monitoring.sh
  ```

🌐 **روابط الدخول الافتراضية للمراقبة:**
* **Prometheus Dashboard:** `http://localhost:9090`
* **Grafana Web Console:** `http://localhost:3001` *(البيانات الافتراضية: admin / admin)*
* **Alertmanager Console:** `http://localhost:9093`

## 🔐 القسم الرابع: إدارة شهادات الأمان والتأمين الرقمي (`deploy/nginx/`)

### 📄 سكريبت تهيئة شهادات الأمان الرقمية (`setup-ssl.sh`)
* **الوصف الفني:** سكريبت مخصص لتوليد وتركيب شهادات SSL رقمية مجانية وموقعة رسمياً من Let's Encrypt لحماية اتصالات المرضى والإدارة (HTTPS) عبر بروتوكول TLS.
* **المدخلات (Inputs):** 
  * اسم النطاق الخاص بالمنصة (Domain Name).
  * البريد الإلكتروني الخاص بالمسؤول لاستقبال إشعارات انتهاء صلاحية الشهادات.
* **المخرجات (Outputs):**
  * تثبيت Certbot والمكونات التابعة لـ Nginx.
  * تعديل ملف إعدادات خادم الويب Nginx تلقائياً ليرتبط بالنطاق الجديد.
  * توليد مفاتيح الشهادات والتحقق من النطاق وتخزينها تحت مجلد `/etc/nginx/ssl/` بصلاحيات أمنية مقيدة (600 للمفاتيح الخاصة).
  * إنشاء سكربت هوك (Hook) تجديد تلقائي آمن يتم استدعاؤه لتحديث خادم الويب فور تجديد الشهادات.
* **أمثلة التشغيل الفعلي:**
  ```bash
  # تشغيل السكريبت بصلاحيات Root لتهيئة وتوليد الشهادات
  sudo chmod +x deploy/nginx/setup-ssl.sh
  sudo ./deploy/nginx/setup-ssl.sh
  ```

🔁 **التحقق من التجديد التلقائي لشهادات SSL:**
تمت تهيئة شهادات الأمان لتتجدد تلقائياً كل 60 يوماً (قبل موعد انتهائها بـ 30 يوماً). يمكن تفعيل فحص محاكاة للتأكد من فاعلية التجديد التلقائي عبر الأمر التالي:
```bash
sudo certbot renew --dry-run
```

لمراقبة حالة اتصال النفق الآمن وصلاحية الشهادات الحالية، يمكن استدعاء سكربت فحص السلامة المرفق:
```bash
./deploy/nginx/verify-ssl.sh
```
